ValidateBox

Override defaults with $.fn.validatebox.defaults.

The validatebox is designed to validate the form input fields. If users enter invalid values, it will change the background color, display the alarm icon and a tooltip message. The validatebox can be integrated with form plugin and will prevent invalid fields from submission.

Validate Rule

The validate rule is defined by using required and validType property, here are the rules already implemented:

  • email: Match email regex rule.
  • url: Match URL regex rule.
  • length[0,100]: Between x and x characters allowed.
  • remote['http://.../action.do','paramName']: Send ajax request to do validate value, return 'true' when successfully.

Properties

Name Type Description Default
required boolean 如果定义字段应该输入。 false
validType string,array 将字段定义有效的类型，如电子邮件，网址，等可能的值是：1）有效的字符串类型，适用于一个单一的验证规则。2）一个有效类型数组，应用多个验证规则。多个验证字段规则可起版1.3.2。代码示例：\r <输入class=“easyui validatebox”数据选项=“要求：真的，validtype：'url”> \r <输入class=“easyui validatebox”数据选项=“\r要求：真的，\r validtype：[ 'email' 'length [ 20 ]”，“] \r” null
delay number 延迟验证从最后一个输入值。此属性是可用的版本以来1.3.2。 200
missingMessage string 显示的工具提示文本时，文本框为空。 This field is required.
invalidMessage string 显示的工具提示文本时，文本框的内容无效。 null
tipPosition string 定义了提示信息的位置文本框的内容是无效的。可能的值：左，右”。此属性是可用的版本以来1.3.2。 right
deltaX number 提示在x方向的偏移。此属性是因为1.3.3版。 0
